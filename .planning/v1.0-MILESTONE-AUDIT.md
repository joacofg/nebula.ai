## Integration Check

**Overall integration status:** passed

### Wiring Summary

- Console admin proxy forwards same-origin browser traffic to `/v1/admin/*` and preserves admin headers and query parameters.
- Playground submits through `/api/playground/completions`, receives `X-Nebula-*` metadata, and performs a follow-up ledger lookup by `request_id`.
- Observability pulls usage-ledger rows through the shared admin client and dependency health through `/api/runtime/health` backed by `/health/dependencies`.
- Policy management uses backend-authored `/policy/options` metadata to drive the editable runtime surface rather than a frontend truth table.
- Phase 5 docs explicitly depend on the benchmark proof package plus the Playground/Observability narrative established by Phases 3 and 4.

### Evidence

- `console/src/app/api/admin/[...path]/route.ts` proxies to `src/nebula/api/routes/admin.py`.
- `console/src/lib/admin-api.ts` calls `/api/admin/policy/options`, `/api/admin/usage/ledger`, and `/api/playground/completions`.
- `console/src/app/(console)/playground/page.tsx` chains immediate response metadata to a recorded ledger lookup.
- `console/src/app/(console)/observability/page.tsx` combines ledger filtering with dependency-health fetches.
- `console/e2e/playground.spec.ts` and `console/e2e/observability.spec.ts` exercise the core operator flows.
