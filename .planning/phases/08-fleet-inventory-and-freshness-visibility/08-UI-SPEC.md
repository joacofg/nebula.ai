## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline pill gaps, dot-to-label gap in dependency pills |
| sm | 8px | Compact element spacing, badge internal padding (`px-2 py-1`) |
| md | 16px | Default element spacing, section `space-y-4`, table cell padding (`px-4 py-4`) |
| lg | 24px | Drawer section spacing (`mt-6 pt-4`) |
| xl | 32px | Major section separation within detail drawer |
| 2xl | 48px | Page-level vertical rhythm between fleet table and controls |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions: Touch targets for action buttons — minimum height enforced via `py-3` (12px top + 12px bottom + 14px line height = ~38px, nearest grid-aligned value to the 40px goal). Drawer close button uses `px-3 py-2` (compact secondary action pattern from existing codebase). Drawer panel padding uses `px-5 py-5` (20px) — inherited from existing `deployment-detail-drawer.tsx` Phase 7 pattern, preserved to avoid visual regression.

**Source:** Existing `deployment-table.tsx` and `deployment-detail-drawer.tsx` establish the `px-4 py-4` table cell rhythm and `px-5 py-5` drawer padding pattern. This phase extends rather than overrides.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label / field label | 12px (text-xs) | 600 (semibold) | 1.0 (single line) |
| Section heading | 12px uppercase tracked (text-xs uppercase tracking-[0.22em]) | 600 (semibold) | 1.0 |
| Deployment name | 20px (text-xl) | 600 (semibold) | 1.2 |

**12px constraint:** 12px at weight 600 (semibold) for all label and heading uses. 12px at weight 400 (regular) is permitted only for monospace data values (version strings, timestamps, dependency names) where font-family distinction (`font-fira-code` vs `font-fira-sans`) provides visual separation from 14px body copy.

**Source:** Detected directly from `deployment-detail-drawer.tsx` and `deployment-table.tsx`. The `text-xs font-semibold uppercase tracking-[0.22em]` pattern is the established "section label" style across the console. Fira Sans body at 14px is the console standard.

Note: The ui-ux-pro-max skill recommended Plus Jakarta Sans for SaaS monitoring dashboards — this is not adopted because the project already has Fira Sans committed across the entire console. Consistency with the existing UI surface outweighs the recommendation.

## Component Inventory

### New Components (Phase 8)

**`freshness-badge.tsx`**
- Props: `status: FreshnessStatus | null`
- When `status` is null: render amber "Awaiting enrollment" badge (same style as pending enrollment state)
- When `status` is defined: render colored badge with label per table above
- Base class: `inline-flex items-center rounded-full border px-2 py-1 text-xs font-semibold`
- Accessibility: Badge text is sufficient; do NOT rely on color alone — the label text ("Connected", "Degraded", "Stale", "Offline") carries the meaning
- Source pattern: matches `DeploymentStatusBadge` structure exactly (from RESEARCH.md Pattern 5)

**`dependency-health-pills.tsx`**
- Props: `summary: HostedDependencySummary | null`
- Renders a single inline row of pills: `[• PostgreSQL] [• Qdrant] [• Ollama]`
- Each pill: colored dot (4px circle, `rounded-full w-2 h-2 flex-shrink-0`) + dependency name at 12px
- Pill container: `inline-flex items-center gap-2 rounded-md bg-slate-100 px-2 py-1`
- All three pills in a `flex flex-wrap gap-2` wrapper
- When `summary` is null: render nothing (pending deployments have no dependency data)
- Accessibility: dependency name text provides the meaning; dot color supplements only

**`freshness.ts` (lib helper)**
- `formatRelativeTime(lastSeenAt: string | null): string` — returns "3 minutes ago", "45 minutes ago", "Mar 20 14:30" (absolute when > 1 hour). Uses `Intl.RelativeTimeFormat` for values under 1 hour.
- `getFreshnessStyle(status: FreshnessStatus | null): { className: string; label: string }` — returns the badge class and display label

### Extended Components (Phase 8)

**`deployment-table.tsx` — column changes**

Current columns: Name, Environment, Status, Enrolled

New columns: Name, Environment, Freshness, Version, Last Seen

- "Status" column removed from table (enrollment state moves to detail drawer per D-11)
- "Enrolled" column removed from table
- "Freshness" column: renders `<FreshnessBadge status={deployment.freshness_status} />`
- "Version" column: renders `<span className="font-[var(--font-fira-code)] text-xs text-slate-600">{deployment.nebula_version ?? "—"}</span>`
- "Last Seen" column: renders relative time string from `formatRelativeTime(deployment.last_seen_at)`; for null: "Never"
- Row opacity: applied via `className` on `<tr>` based on freshness status — stale rows get `opacity-75`, offline rows get `opacity-60`, all others unchanged
- Row selection highlight: remains `bg-sky-50` for selected row (unchanged)
- Pending (never enrolled) rows: normal opacity, freshness badge shows "Awaiting enrollment"

**`deployment-detail-drawer.tsx` — section additions**

Current layout: header → identity fields → lifecycle actions

New layout (per D-12):
1. Header (unchanged: eyebrow label + deployment name + close button)
2. **Freshness section** (NEW — inserted before identity fields)
   - `<FreshnessBadge />` at full-width (not inline)
   - Reason text: 14px regular slate-600, e.g. "Last heartbeat 3 minutes ago"
   - Last-seen timestamp: 12px monospace slate-400 (absolute formatted date)
   - Section divider: `mt-4 border-t border-border/50 pt-4` before next section
3. **Identity section** (existing fields, no visual change)
   - Deployment ID (unchanged)
   - Environment (unchanged)
   - Nebula version — already present, keep as-is
   - **Capability flags** (already present as chips) — keep existing chip style: `rounded-md bg-slate-100 px-2 py-1 font-[var(--font-fira-code)] text-xs text-slate-600`
   - Enrolled at (unchanged)
   - Created at (unchanged)
   - **Enrollment state** (NEW field added here — moved from table per D-11): renders `<DeploymentStatusBadge />`
4. **Dependency health section** (NEW)
   - Section label: "Dependencies" in `text-xs font-semibold uppercase tracking-[0.22em] text-slate-500`
   - Renders `<DependencyHealthPills summary={deployment.dependency_summary} />`
   - When null: render `<span className="text-sm text-slate-400">No data reported</span>`
5. **Trust-boundary disclosure card** (NEW — imported from Phase 6)
   - `import { TrustBoundaryCard } from "@/components/hosted/trust-boundary-card"`
   - Renders as-is from Phase 6 implementation, no modifications needed
6. **Lifecycle actions** (unchanged position — remains at bottom)

## Layout Contract

### Fleet Table Layout

```
[Page header]
  [Left: "Deployments" page heading — dominant visual anchor (text-2xl font-semibold)]
  [Right: "Register deployment" CTA button — secondary element, accent color]
[Fleet table panel]
  thead: Name | Environment | Freshness | Version | Last Seen
  tbody: one <tr> per deployment, clickable
[Detail drawer: right panel, appears when row selected]
```

The page heading ("Deployments") is the dominant visual anchor — largest text on the page at 20px semibold. The "Register deployment" CTA is a secondary element positioned in the header row at the same vertical level but visually subordinate: smaller, right-aligned, accent-colored button. Operators scan the heading first to confirm page context, then locate the CTA when they need to act.

Table is a full-width `<table>` inside a `.panel` container with `overflow-x-auto` wrapper (unchanged from existing pattern).

## Accessibility Contract

- Freshness badges: status conveyed by text label, not color alone. Badge text ("Connected", "Degraded", "Stale", "Offline") is the primary indicator. Color is supplemental. WCAG 1.4.1 compliant.
- Dimmed rows (stale at opacity-75, offline at opacity-60): text within dimmed rows remains `text-slate-950` / `text-slate-600` to preserve contrast ratios above 4.5:1 against the white table background.
- Dependency health pills: dot color is supplemental; dependency name text is always present. WCAG 1.4.1 compliant.
- Error messages on fetch failure: rendered with `role="alert"` so screen readers announce them. WCAG 4.1.3 compliant.
- Disabled action buttons: `disabled` attribute applied (not just `opacity-60`). This ensures screen readers announce the disabled state and the button is removed from tab order.
- All interactive rows: `cursor-pointer` applied. `hover:bg-slate-50` transition at `transition-colors duration-150` (150ms, under the 200ms threshold).
- `prefers-reduced-motion`: existing `globals.css` rule already covers all transitions with `animation-duration: 0.01ms`. No additional work needed.
