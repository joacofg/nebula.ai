    **5. Create `src/nebula/services/gateway_remote_management_service.py`** with class `GatewayRemoteManagementService` and these exact methods:
    - `start() -> None`
    - `stop() -> Awaitable[None]`
    - `_loop() -> Awaitable[None]`
    - `_poll_once() -> Awaitable[None]`
    - `_is_action_locally_allowed(action_type: str) -> tuple[bool, str | None]`
    - `_apply_rotate_deployment_credential(action_id: str) -> Awaitable[None]`

    Polling rules:
    - Skip silently if no active local identity or no hosted plane URL
    - Poll `GET {hosted_plane_url}/v1/remote-actions/next` every `remote_management_poll_interval_seconds`
    - Authenticate with the existing `X-Nebula-Deployment-Credential` header
    - On `204`, do nothing
    - On `200`, parse a `RemoteActionRecord`

    Local authorization rules in `_is_action_locally_allowed`:
    - If `settings.remote_management_enabled is False`, return `(False, "unauthorized_local_policy")`
    - If `action_type` not in `settings.remote_management_allowed_actions`, return `(False, "unauthorized_local_policy")`
    - If `action_type == "rotate_deployment_credential"` and `"remote_credential_rotation"` not in `_derive_capability_flags(settings)`, return `(False, "unsupported_capability")`

    **6. Create `src/nebula/api/routes/remote_management.py`** with `remote_management_router = APIRouter(prefix="/remote-actions", tags=["remote-management"])` and these exact endpoints:
    - `GET /remote-actions/next`
    - `POST /remote-actions/{action_id}/claim-rotation`
    - `POST /remote-actions/{action_id}/ack`

    Endpoint behavior and service methods on `RemoteManagementService`:
    - `next_remote_action(raw_credential: str | None) -> RemoteActionRecord | None`
      returns the oldest queued live action for the deployment authenticated by the current credential; returns `None` if none
    - `claim_rotate_deployment_credential(action_id: str, raw_credential: str | None) -> str`
      under lock: validate the action belongs to the authenticated deployment, is `status=="queued"`, not expired, and has no terminal timestamp;
      generate a new raw credential with prefix `nbdc_`,
      store only `pending_credential_hash` and `pending_credential_prefix` on the action row,
      set `claimed_at=now`,
      return the raw new credential in the response body
    - `ack_remote_action(action_id: str, raw_credential: str | None, outcome: str, failure_reason: str | None, failure_detail: str | None) -> RemoteActionRecord`
      if `outcome=="applied"`, promote `pending_credential_hash` and `pending_credential_prefix` into the `DeploymentModel`, set `status="applied"`, `applied_at=now`, clear pending fields
      if `outcome=="failed"`, set `status="failed"`, `failed_at=now`, `failure_reason`, and `failure_detail`, clear pending fields
      keep authentication on the old credential for this ack path; rotate the deployment row only inside the same transaction that marks `applied`

    **7. In `GatewayRemoteManagementService._poll_once()` implement this exact behavior:**
    - Poll `GET /v1/remote-actions/next`
    - If the action type is not locally allowed, immediately `POST /v1/remote-actions/{id}/ack` with JSON:

      `{"outcome":"applied","failure_reason":null,"failure_detail":null}`
    - If local credential persistence throws, catch it and ack failed with `failure_reason="apply_error"` and the exception string in `failure_detail`
