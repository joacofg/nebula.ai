# Phase 7 — UI Design Contract

> Visual and interaction contract for the deployment enrollment and identity console surface.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom Tailwind + CSS custom properties |
| Preset | not applicable |
| Component library | none (hand-rolled components using global utility classes: `.panel`, `.field-input`, `.field-label`, `.action-button`, `.secondary-button`) |
| Icon library | lucide-react (already in project) |
| Font — body | Fira Sans (`--font-fira-sans`), loaded via `next/font/google` |
| Font — mono | Fira Code (`--font-fira-code`), used for tokens, IDs, and code values |

Source: `console/src/app/globals.css`, `console/src/app/layout.tsx`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, bullet dot spacing, inline separators |
| sm | 8px | Compact element spacing, tight label-to-input gap |
| md | 16px | Default element spacing, form field vertical gap (`space-y-4`) |
| lg | 24px | Section padding, drawer interior padding (`px-5 py-5`) |
| xl | 32px | Layout column gap (`gap-6` ≈ 24px; outer section spacing `space-y-6`) |
| 2xl | 48px | Major section breaks (page header to content) |
| 3xl | 64px | Page-level vertical padding |

Exceptions:
- Copy-to-clipboard token block: `px-4 py-4` (16px) to match `RevealApiKeyDialog` dark slab pattern.
- Status badge pills: `px-2 py-0.5` (8px / 2px) — intentionally compact.
- Icon touch targets in table rows: minimum 44px tall row height via `py-3` on `<td>`.

## Color

| Role | CSS Custom Property | Resolved Value | Usage |
|------|---------------------|----------------|-------|
| Dominant (60%) | `--color-background` | `rgb(248 250 252)` — slate-50 | Page background, section backgrounds |
| Secondary (30%) | `--color-surface` | `rgb(255 255 255)` — white | `.panel` cards, drawers, dialogs |
| Accent (10%) | `--color-accent` | `rgb(3 105 161)` — sky-700 | Reserved elements listed below |
| Danger | `--color-danger` | `rgb(190 24 93)` — pink-700 | Revoke action button and confirmation only |

Source: `console/src/app/globals.css`.

Accent (`rgb(3 105 161)`) is reserved for:
1. `.action-button` background (primary CTAs: "Create deployment slot", "Generate enrollment token", "Relink deployment")
2. Focus ring on `.field-input` (`focus:ring-accent/20`)
3. Eyebrow label color (`text-sky-700`)
4. Status badge for `active` state (`bg-sky-50 text-sky-700 border-sky-200`)
5. Bullet dot for deployed/active list items (`bg-sky-500`)
6. Token slab text (`text-sky-100` on `bg-slate-950`)

Danger (`rgb(190 24 93)`) is reserved for:
1. "Revoke" destructive action button background (styled as `.action-button` override: `bg-pink-700 hover:bg-pink-800`)
2. Confirmation dialog border and heading tint

Additional semantic colors for status badges (read-only, no interactive use):
- `pending`: `bg-amber-50 text-amber-700 border-amber-200`
- `active`: `bg-sky-50 text-sky-700 border-sky-200`
- `revoked`: `bg-rose-50 text-rose-700 border-rose-200`
- `unlinked`: `bg-slate-100 text-slate-600 border-slate-200`

## Page Layout

The deployment management page follows the established split-panel pattern from `console/src/app/(console)/tenants/page.tsx`:

```
<section class="space-y-6">
  <header class="panel ...">  ← page header: eyebrow + heading + description + CTA
  <div class="grid xl:grid-cols-[minmax(0,1.7fr)_minmax(320px,0.95fr)]">
    ← left: search/filter bar + DeploymentTable
    ← right: CreateDeploymentSlotDrawer / DeploymentDetailDrawer
  </div>
</section>
```

Modal overlays (EnrollmentTokenRevealDialog, RevokeConfirmationDialog) render at `z-40` with `bg-slate-950/45 backdrop-blur-sm`.

## States and Interactions

### Deployment Table Row

| State | Visual Treatment |
|-------|-----------------|
| No deployments yet | Empty state panel with heading + body copy + CTA (see Copywriting) |
| Loading | Single `.panel` with muted loading copy (matches tenant page pattern) |
| Load error | `.panel` with `border-rose-200 bg-rose-50 text-rose-900` error copy |
| Row selected | Highlighted row (`bg-sky-50`) with drawer open on right |
| Pending row | Status badge `pending` (amber); no action menu items except "Copy token" if token not yet consumed |
| Active row | Status badge `active` (sky); action menu: "Unlink", "Revoke" |
| Revoked row | Status badge `revoked` (rose); action menu: "Relink" |
| Unlinked row | Status badge `unlinked` (slate); action menu: "Relink" |

### Enrollment Token Generation

After "Create deployment slot" succeeds, the `EnrollmentTokenRevealDialog` opens immediately with:
1. Icon: `KeyRound` (Lucide) in `bg-sky-50 text-sky-700` rounded square (matches reveal-api-key pattern)
2. One-time disclosure warning: see Copywriting
3. Dark slab with token string in Fira Code
4. "Copy token" button (accent) + "Close" button (secondary)
5. Env var instruction block (see Copywriting)

The dialog does not close automatically. Operator must click "Close" explicitly.

### Revoke Confirmation

Revoke opens a modal overlay with:
1. Danger heading (no icon required)
2. Consequence description: immediate credential invalidation
3. "Revoke deployment" button (`bg-pink-700 hover:bg-pink-800 text-white`)
4. "Cancel" button (`secondary-button`)
5. No text-entry confirmation required (follows existing project pattern — no "type to confirm" gate)

### Unlink Confirmation

Unlink opens the same modal overlay pattern but uses the secondary button as the confirm action (not danger-styled, since unlink is non-destructive):
1. Neutral heading
2. Consequence description: gateway continues serving, loses hosted features
3. "Unlink deployment" button (`secondary-button`)
4. "Cancel" button (`secondary-button`)

## Copywriting Contract

### Primary CTAs

| Element | Copy |
|---------|------|
| Page header CTA | "Create deployment slot" |
| Form submit — create | "Create deployment slot" |
| Token reveal — copy | "Copy token" / "Copied" (toggle after clipboard write) |
| Relink action | "Generate new token" |
| Revoke confirm button | "Revoke deployment" |
| Unlink confirm button | "Unlink deployment" |

### Empty State

| Element | Copy |
|---------|------|
| Empty state heading | "No deployments linked" |
| Empty state body | "Create a deployment slot to register a self-hosted Nebula gateway. You'll receive a single-use enrollment token to paste into your gateway's environment config." |
| Empty state CTA | "Create deployment slot" |

### One-Time Token Disclosure

| Element | Copy |
|---------|------|
| Dialog heading | "Enrollment token" |
| Disclosure warning | "This token will not be shown again. Copy it now and set it as NEBULA_ENROLLMENT_TOKEN in your gateway's environment, then restart the gateway." |
| Env var instruction | `NEBULA_ENROLLMENT_TOKEN=<token>` (rendered in enrollment instructions block) |
| Expiry note | "Token expires in 1 hour. If it expires before use, generate a new one from this deployment's detail panel." |

### Error States

| Scenario | Copy |
|----------|------|
| Load error (deployment list) | "Unable to load deployments. Check your operator session and try reloading." |
| Create slot error | "Unable to create deployment slot. [server message if available]" |
| Generate token error | "Unable to generate enrollment token. Try again or reload the page." |
| Revoke error | "Unable to revoke this deployment. Refresh and try again." |
| Unlink error | "Unable to unlink this deployment. Refresh and try again." |

### Destructive Action Confirmations

| Action | Confirmation Copy |
|--------|------------------|
| Revoke | Heading: "Revoke this deployment?" Body: "The deployment's credential will be invalidated immediately. The gateway will lose hosted-plane access on its next heartbeat. Use Relink to restore access." |
| Unlink | Heading: "Unlink this deployment?" Body: "The gateway will stop sending heartbeats and lose hosted features. Local serving continues unaffected. You can relink at any time." |

### Status Labels (read-only badges)

| State | Label | Color treatment |
|-------|-------|-----------------|
| `pending` | "Pending enrollment" | amber |
| `active` | "Active" | sky/accent |
| `revoked` | "Revoked" | rose |
| `unlinked` | "Unlinked" | slate |

### Page Header

| Element | Copy |
|---------|------|
| Eyebrow | "Deployments" |
| Heading | "Deployment management" |
| Description | `Manage linked self-hosted Nebula deployments through /api/admin/deployments.` (Fira Code for endpoint, matching tenant page pattern) |

### Trust Boundary Reminder (inline, below page header)

The deployment creation form includes a single-sentence trust reminder above the form fields (not a full card — this is covered by Phase 6):

> "Linking is outbound-only. The hosted plane receives metadata only — no prompts, responses, or provider credentials."

Rendered as `text-xs text-slate-500 mt-1` below the form description paragraph. Sourced from Phase 6 CONTEXT.md decision on disclosure surfaces.
