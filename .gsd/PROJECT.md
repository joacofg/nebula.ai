# Project

## What This Is

Nebula is a self-hosted semantic AI gateway for teams that want to reduce premium LLM spend without losing control over routing, fallback behavior, or operator visibility. v1.0 and v2.0 are already complete. v3.x established Nebula's adoption and hosted-reinforcement story. v4.x turned Nebula’s existing evidence, policy, routing, and cache surfaces into a stronger decisioning control plane, M007 clarified the operator console into a more coherent request-led decision system, and M008 completed Nebula’s bounded evidence-governance and privacy-controls milestone.

## Core Value

A team can adopt Nebula through a familiar inference path quickly, with minimal application changes, while gaining routing, policy, observability, provider abstraction, and interpretable operator evidence that stays under local operator control.

## Architecture / Key Patterns

Nebula routes `POST /v1/chat/completions` traffic across local, cache, premium, and fallback paths, returning `X-Nebula-*` metadata headers and persisting usage outcomes to the governance ledger. The backend is FastAPI with a service-container pattern; the operator console is Next.js and proxies same-origin admin traffic to gateway APIs. Governance is tenant-centric today. The hosted control plane is metadata-only by default and is not authoritative for local runtime enforcement. Existing decisioning work favors stable typed evidence, bounded operator-facing read models, request-led operator proof, and pointer-only integrated proof docs over duplicated contracts or opaque optimization claims.

M008 extends those patterns by keeping evidence governance inside the existing policy and ledger boundaries. The central write-time enforcement seam remains `GovernanceStore.record_usage()`, which applies tenant retention/minimization policy at write time for both chat and embeddings and persists historically truthful governance markers on each ledger row for later inspection. The central deletion seam remains `GovernanceStore.delete_expired_usage_records(now=...)`, and S03 adds the operational lifecycle around it through `RetentionLifecycleService`, which runs as a bounded background service, records in-memory execution diagnostics for health surfaces, and keeps cleanup proof on existing `/health/ready`, `/health/dependencies`, observability dependency cards, and `/v1/admin/usage/ledger` before/after inspection seams instead of inventing a retention-specific API or dashboard. S04 adds a parallel UI copy pattern: `console/src/lib/hosted-contract.ts` is the shared evidence-vocabulary contract for retained/suppressed/deleted/not-hosted wording and hosted raw-ledger exclusion copy, which the policy form, ledger request detail, hosted trust card, and public trust-boundary page all reuse so operator-facing evidence semantics stay aligned with the metadata-only hosted boundary. S05 closes the milestone with a final assembly pattern: integrated proof docs remain pointer-only, request-led evidence stays primary, runtime health remains supporting context, and hosted wording remains explicitly metadata-only and non-authoritative.
