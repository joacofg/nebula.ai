# T02: Added a tenant-scoped admin recommendations endpoint, shared console contract typings, and focused governance API coverage for bounded cache evidence.

**Added a tenant-scoped admin recommendations endpoint, shared console contract typings, and focused governance API coverage for bounded cache evidence.**

## What Happened

Added a single admin-only read endpoint at `/v1/admin/tenants/{tenant_id}/recommendations` that checks tenant existence and returns the existing bounded `RecommendationBundle` generated by the backend recommendation service. Extended `console/src/lib/admin-api.ts` with shared recommendation/cache summary types plus a `getTenantRecommendations` helper so downstream UI surfaces can consume the same contract without ad-hoc parsing. While verifying the route, governance API startup failed before the new endpoint executed because the test database schema was missing the semantic cache tuning columns introduced in T01. I confirmed the root cause by comparing the ORM model and Alembic revisions, then added an idempotent migration (`20260328_0008`) to add `semantic_cache_similarity_threshold` and `semantic_cache_max_entry_age_hours`. I also fixed the governance API test helper so cache-service overrides propagate into `recommendation_service`, which made degraded-cache evidence tests deterministic instead of accidentally consulting the live semantic cache dependency. Finally, I added focused tests covering admin protection, explicit 404 behavior for missing tenants, tenant scoping, bounded response structure, and degraded cache evidence propagation.

## Verification

Ran `./.venv/bin/pytest tests/test_governance_api.py -k "recommendation or cache" -x` until green after fixing the missing migration and test harness dependency injection. Final run passed with 6 selected tests covering admin protection, 404 handling, tenant scoping, bounded response structure, and degraded cache evidence propagation. Also ran LSP diagnostics on `src/nebula/api/routes/admin.py`, `tests/test_governance_api.py`, and `migrations/versions/20260328_0008_cache_policy_controls.py`; all returned clean results.
